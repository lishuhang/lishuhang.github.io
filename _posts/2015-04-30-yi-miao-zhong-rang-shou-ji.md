微软昨天举行了 Build 开发者大会。在开发者大会上，有几个非常重要的宣布，将会决定未来一段时间 Windows 以及 WP 的走向。

**【Universal Apps：手机电脑二合一】**

微软的第一个大招是手机将会直接变身“电脑”，通过线缆把手机连接到屏幕上之后，它上面的 Universal Apps 会自动放大到适应整个屏幕，那样就像是在一台电脑上办公。

我一直有个疑问，现在手机的机能已经赶上电脑，不知道为什么，手机接到显示器上的时候感觉总是和电脑不一样。如果 Office 等应用都可以从你的口袋里“变”出来，那会让手机的实用性大为提高。

此前已经有人众筹口袋大小的电脑盒子，需要外接显示器显示，但里面运行的是 PC 操作系统。那种设备的出现证明了人们有更便携地迁移自己办公环境的需求。也就是说，不带公文包，就把手机装口袋，到办公室接显示器和键盘鼠标办公的情景也不再是天方夜谭。

进一步的依赖 Universal Apps（过去我们称为 Metro 界面和 Modern 界面，微软改名部辛苦了）将会使得微软淘汰旧式 x86 应用的进度更加加快。

**【兼容 iOS 和 Android Apps】**

第二个大招是可以将 iOS 和 Android 上的软件移植到 Windows 平台上。目前还不清楚的就是是否最终也可以在 PC 版 Windows 10 上面安装，毕竟除了 Universal Apps，给手机和给 PC 的 Windows 10 还有很多不同。但至少现在的情况是：

> Windows 10 PC = Universal Apps + x86 Apps
> Windows 10 Phone = Universal Apps + 移植的 iOS / Android Apps

iOS / Android Apps 不能直接被转化为 Universal Apps，感觉上就像是跑虚拟机一样。虽然微软指出，在移植过程中，相关接口都会被转写成 Windows 中对应的接口，但是最终为了确保应用体现出来的效果一致，以及其中可能使用的第三方引擎如 Unity 工作正常，可能还是要使用虚拟机类运行的办法。这样，应用工作的效率可能就没有原生的那么高。

对于开发者而言，他们有可能会使用 Visual Studio 用一套源码直接输出 iOS 和 Android Apps。能够实现一套代码跨平台的编程工具其实现在已经有很多种，包括 Adobe Flash 都借尸还魂，它也可以通过 AIR 生成 iOS、Android 和桌面程序。但是大家都还是喜欢各自的原生编程，无他，还是运行效率作怪。

**【Spartan 定名为 Edge】**

对于普通用户而言值得注意的是，微软将开发代号为 Spartan 的浏览器定名为 Edge。这个名称的推出有可能又要麻烦微软改名部忙一阵子。

微软之前并不是没有犯过名字起错了的低级错误，比如说 SkyDrive 和 Metro UI 都是因为没有仔细的查找商标重复而导致官司，最终被迫改为其他的名字。在我的印象中，至少 Ubuntu 就使用过 Edge 作为手机的名称。

非常有趣的是，Ubuntu Edge 这个在 Indiegogo 众筹的产品，一个主打功能恰好是我们上面谈到的第一点：把手机接到显示器上，当电脑来运行！只不过因为系统是 Ubuntu，应用面太窄，所以一直不温不火而已。

![](https://lishuhang.me/img/2015/04/30/yi-miao-zhong-rang-shou-ji/01.jpg)

Edge 的 logo 是跟 ie 标志长得差不多的，并获得了如此评价：

> 纳得拉说 IE 太大了，该卸下点东西。于是改名部隔壁的美工部首当其冲，拿下了原来 IE 图标里一个圆环和一个扇形。

看来微软也非常懂得中国一系列大大小小的壳浏览器的生存之道，就是让自己长得更像 ie。而微软能够从中国的壳浏览器当中得到的另外一个启示就是，只要不是根本运行不了，或者麻烦太多问题太大，用户根本不会在意他们用的是什么东西。

**【结论】**

即使距离七月份微软最终发布正式版 Windows 10 的日子已经越来越近，预览版依然问题多多，时不时崩溃；而微软寄予厚望的 Universal Apps 在实际操作中运行效率极低，远远低于 x86 应用，机器越老就会面临越多的兼容性问题。但是，这些大的麻烦似乎只能成为遗憾，系统的功能和界面应该基本定型，剩下的只是调整和修复漏洞的问题。但是别小看最后冲刺，因为也许为了修补漏洞和让问题不再发生，还会有其他的一些功能面临修改或去掉的命运。

在本次 Build 大会之后，Windows 10 可能将史无前例地同时允许三个不同架构下的应用运行——x86 Apps、 Universal Apps 和移植的 iOS / Android Apps。三者不管是运行原理，软件构成还是界面都完全不相同，但微软似乎因为 Windows 三十多年来的兼容史，对解决如此厚重的兼容性问题还充满信心。

Windows XP 此刻依然运行在全球不知多少的电脑上，人们对于 PC 的使用心态是复杂而纠结的。如果 Windows 10 这一仗不能打赢的话，PC 使用者的分化将会更为严重，对这个复杂的烂摊子，微软将更没有办法管理。所以，让我们和微软官方一样，都翘首期盼着 Windows 10 能够好好的处理以上所有问题，给我们一个让人眼前一亮的答案。
