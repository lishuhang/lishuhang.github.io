Internet Explorer 在稍微有一些网络知识的互联网用户当中具有很恶劣的评价。微软对最近的几个版本 IE 进行了大量宣传，包括进军社交网络，甚至推出针对亚洲市场的萌化卡通形象，但依然收效甚微。

为什么有那么多你身边的人讨厌 IE ？为什么不讨厌 IE 的人依然是“不管用什么浏览器都无所谓”的类型？为什么我们很少看见有人主动跑上来说：我就是喜欢 IE ？

**1、为之前的“垄断”印象还债**

IE 的发家史很多人都已经了解，微软是通过在操作系统中捆绑打败了 Netscape ，确立了长期的垄断地位，以及由此生成的不思进取的印象。随后出现的 Firefox 和 Chrome 则被视作类似苹果“1984”广告中的颠覆“老大哥”的挑战者。

当然，就像做出“1984”的苹果在今天已经向着“老大哥”本身蜕变一样，Firefox 和 Chrome 也难说自己就白璧无瑕，但它们针对 IE 的刻板印象已经形成——Firefox 的刻板印象是功能很多，插件很多，速度很慢；Chrome 则开局良好，让人“看起来”不那么臃肿。这跟它实际上臃肿不臃肿是两码事儿。比较页面加载速度、内存占用等，这三款桌面浏览器都已经没有特别明显的优劣之分。

随着 Windows 和 Windows Phone 在通用计算操作系统当中的份额开始让位给 Mac、iOS、Android 等，IE 即使可以在Windows当中捆绑，也要应付 Safari、Chromium 等其他系统捆绑浏览器的袭击。而对方的浏览器捆绑系统则是生来“纯净”，在大众印象和法庭判例中都不带有垄断的“原罪”。前几年欧盟还专门找微软的茬，搞得欧盟的 Windows 得推出一个浏览器选单，让微软闹心不已。

**2、仍然希望“改正”而不是遵循用户的原有使用习惯**

要说简洁？目前这个星球上最简洁的浏览器非 IE 莫属。从 9.0 版本开始，IE 已经向着现代浏览器的方向迈出了关键的一步。IE 9 的官方卖点就是“界面设计简洁，并与 Windows 7 良好集成，为网站提供更大的显示区域，从而让用户聚焦于应用本身，而不是浏览器。浏览器控件减少并且处于半透明状态，避免干扰用户的视线，真正让网络内容成为用户关注的焦点。”然而，它似乎从未得到跟这种卖点相一致的评价。

我想 IE 9 以后一个新的设计思路已经非常明显，那就是不打算继续加入和维护任何类型的插件功能（不管是 IE 6 时代的工具条，还是 ActiveX 控件，还是 Firefox / Chrome 插件，Greasemonkey 脚本，甚至 Web Apps），即使桌面用户对插件很渴望。

在 IE 看来，现在浏览器除单纯浏览之外唯一的用途就是离线存储和通知功能。这些功能都可以通过 HTML5 来实现。IE 还专门为此加入了 Win 7 任务栏固定和 Win 8 开始菜单磁贴，将网页变成“伪应用”。如果用户仅仅是想浏览网页而已，这些功能的确已经足够。

但是使用桌面版浏览器的人，有时需要更改浏览器的 User-agent，有的时候需要切换代理服务器，有的时候需要嗅探和下载网页视频，有的时候需要批量保存图片，或者为页面截图，还有人可能想要一个手势。插件的方便性，暂时还是无可替代的。

就像 Windows 8 的激进改动引发用户反弹一样，微软试图使用尚高的市场占有率，来强行教育用户学习新的操作习惯，但是在备选方案增多的今天，微软的举动不仅引发了自身市场份额的流失，更严重的是推动了 PC 平台整体的衰落。IE 也是试图“教育用户”不用插件，但效果不佳的案例。

**3、糟糕的开发者社区建设**

在其他桌面浏览器纷纷推出插件功能的时候，微软始终对此持保留态度，他们唯一一次努力的想建设插件体系，并向开发者示好是在 IE 7 到 IE 8 的阶段。这两个版本 IE 横跨 Vista 和 Windows 7 两个系统，创建了包含搜索提供商，智能标签，和传统的网页工具栏在内的插件中心，并且允许普通用户免费提交作品。

那个时候也正好赶上微软要把以前的 msn 整合成 Windows Live ，同时也引入了小工具（Gadgets），这也是微软空前，而且很可能绝后的行动。但是这种社区最终只是昙花一现。我们看到微软的 Live 战略完全失败，在 Windows 8 时代推倒重来。

就算微软对普通开发者社区再缺乏感情，反应迟钝，毕竟也有上千款独立插件和 Windows 小工具被爱好者做出来，几乎形成了一个有希望跟后来者 Chrome 应用商店在规模上匹敌的市场。随着 IE 9 开始推行网站固定任务栏、动态磁贴等新功能，淡化智能标签等原有插件功能，这些插件被打入冷宫。因为具有安全隐患的官方说法，上千款小工具在新版系统中无法使用。

微软其实有能力加固小工具的安全系数，但安全更多只是一个借口。微软更愿意推动动态磁贴功能，而这种功能的背后并非是爱好者的社群，而是变成了开发者上传-审核-分成的具有严格规范的应用商店。其操作思路跟运营一个社区是不一样的。

种种事实都向我们表明，微软其实并没有运作一个良好社区的能力。所以，其他浏览器由用户创建并维护的万用功能，从来就不是微软的长项。

**4、成为“山寨”标杆**

预装到电脑里面，而且无法卸载的 IE 获得粉丝的概率，肯定小于人们自愿下载并且选择使用的浏览器。愿意发出声音说 IE 以外的浏览器很好的用户，都对自己选用 IE 之外的浏览器这一点有着充分的自觉，他们可能使用 IE 做的唯一一件事情，就是去下载其他的浏览器。

对这些用户而言，他们看到自己不懂电脑的家人朋友，在桌面上并排出现绿色、红色、蓝色的很多个“e”字图标，可能唯一的感受就是抓狂。是的，在中国疯狂崛起的壳浏览器，最喜欢使用的图标造型就是 IE 的字母“e”，并在设计上尽量和 IE 靠拢，以混淆用户的双眼。

包括网页设计师在内的 IT 专业人士，从 IE 6 时代就对 IE 忍无可忍，“山寨”印象更加剧了这种矛盾对立。他们发起一系列在页面加插标记，要求用户更新浏览器的行动，将 IE 描写为一个脆弱，老朽，容易中病毒的浏览器，甚至塑造“使用火狐和谷歌浏览器不会中毒”的印象。

大概从 2006 年左右开始的这个挤兑 IE 的行动，最终结果是被大型“山寨”浏览器借用，变成了“升级系统自带浏览器，使用安全绿色无病毒的XXX安全浏览器”，完全变成了一地鸡毛。只有 IE 成为各家同时借用的良好的靶子。

**结论**

纵观微软最近一段时间对 IE 的宣传行动，可以说主要集中在以下几个点：一是怀旧，表明当年 IE 是给大家带来网络启蒙的老师。二是速度和性能比拼，微软积极推进新的网页标准和设计规范，甚至比其他浏览器还要稍微激进一点。三是专门针对 Windows 8 平板的用户宣传触摸新功能，让他们从零开始培养浏览习惯。

但是根据上面所说的几点，可以说，这些宣传战役是完全没有触及要害。加入与操作融合更自然的插件平台，不再试图“教育用户”将桌面和平板浏览体验统一，可能是更好的开始，这会让有心回到 IE 的用户不必因为使用上的不习惯望而却步。
